## 🚀 快速启动命令

### 启动 Redis
```bash
# 启动 Redis (后台运行)
redis-server --daemonize yes

# 测试 Redis 连接
redis-cli ping
# 应该返回: PONG

# 停止 Redis
redis-cli shutdown
```

### 管理 MySQL

#### 选项1: 使用现有的旧版 MySQL
```bash
# 连接到现有 MySQL (无密码)
mysql -u root

# 如果有密码,使用:
mysql -u root -p
```

#### 选项2: 切换到 Homebrew 安装的新版 MySQL

```bash
# 1. 停止旧版 MySQL
sudo launchctl unload -w /Library/LaunchDaemons/com.oracle.oss.mysql.mysqld.plist

# 2. 启动 Homebrew MySQL
brew services start mysql

# 3. 连接到新版 MySQL
mysql -u root

# 4. 设置 root 密码(可选)
mysql_secure_installation
```

## 🔧 服务管理命令

### Redis 命令

```bash
# 启动
redis-server --daemonize yes

# 停止
redis-cli shutdown

# 查看状态
redis-cli ping

# 连接到 Redis CLI
redis-cli

# 查看所有键
redis-cli KEYS "*"

# 清空所有数据
redis-cli FLUSHALL
```

### MySQL 命令

```bash
# 启动 (Homebrew 版本)
brew services start mysql

# 停止
brew services stop mysql

# 重启
brew services restart mysql

# 查看状态
brew services list

# 连接
mysql -u root

# 查看进程
ps aux | grep mysqld
```

## 🐛 常见问题

### 1. Redis 连接失败
```bash
# 检查 Redis 是否运行
redis-cli ping

# 如果没有运行,启动它
redis-server --daemonize yes
```

### 2. MySQL 端口冲突
```bash
# 检查谁在使用 3306 端口
lsof -i :3306

# 停止旧版 MySQL
sudo launchctl unload -w /Library/LaunchDaemons/com.oracle.oss.mysql.mysqld.plist
```

### 3. 数据库连接被拒绝
```bash
# 确保 MySQL 正在运行
brew services list | grep mysql

# 重置 root 密码
mysql_secure_installation
```

## 📊 查看服务状态

```bash
# 一键查看所有服务状态
echo "=== MySQL 状态 ==="
ps aux | grep mysqld | grep -v grep
echo ""
echo "=== Redis 状态 ==="
redis-cli ping 2>/dev/null && echo "Redis: ✅ 运行中" || echo "Redis: ❌ 未运行"
echo ""
echo "=== 端口占用 ==="
echo "MySQL (3306):"
lsof -i :3306 | head -2
echo "Redis (6379):"
lsof -i :6379 | head -2
```

## 🎯 推荐开发流程

1. **启动本地服务**
   ```bash
   redis-server --daemonize yes
   # MySQL 已经在运行,无需启动
   ```

2. **修改配置文件**
   - 编辑 `backend/.env`
   - 将数据库和 Redis 改为本地连接

3. **创建数据库**
   ```bash
   mysql -u root -e "CREATE DATABASE IF NOT EXISTS bitcoin_mining_master CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
   ```

4. **运行迁移**
   ```bash
   cd backend
   npx sequelize-cli db:migrate
   ```

5. **启动项目**
   ```bash
   pm2 start ecosystem.config.js
   ```

6. **开发完成后提交代码**
   ```bash
   git add .
   git commit -m "本地开发完成"
   git push
   ```

---

## ⚠️ 注意事项

1. **不要提交 .env 文件** - 已在 .gitignore 中排除
2. **本地和远程数据分离** - 本地测试用本地数据库,生产用远程
3. **定期备份** - 本地数据库也要定期备份
4. **密码安全** - 本地开发可以无密码,生产环境必须设置强密码
