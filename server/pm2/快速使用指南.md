## 🔧 其他实用命令

### 查看服务状态
```bash
ssh root@47.79.232.189 "pm2 status"
```

### 查看日志
```bash
# 查看最新日志
ssh root@47.79.232.189 "pm2 logs bmm-api --lines 20 --nostream"

# 查看错误日志
ssh root@47.79.232.189 "pm2 logs bmm-api --err --lines 50 --nostream"

```

### 健康检查
```bash

curl http://47.79.232.189:8888/api/health

# 在服务器上测试
ssh root@47.79.232.189 "curl -s http://127.0.0.1:8888/api/health"
```

### 重启特定服务
```bash

```

### 查看详细信息

## 📋 SSH 配置优化（可选）

创建 SSH 配置文件，简化命令：

```bash
cat > ~/.ssh/config << 'EOF'
Host bmm
    HostName 47.79.232.189
    User root
    IdentityFile ~/.ssh/id_ed25519
    ServerAliveInterval 60
    ServerAliveCountMax 3
EOF

chmod 600 ~/.ssh/config
```

配置后可以简化命令：
```bash
# 原来：ssh root@47.79.232.189
# 现在：ssh bmm

# 原来：ssh root@47.79.232.189 "pm2 list"
# 现在：ssh bmm "pm2 list"
```

## ⚠️ 如果 ssh-copy-id 失败

### 方法 1：手动添加公钥

```bash
# 1. 复制你的公钥
cat ~/.ssh/id_ed25519.pub

# 2. SSH 登录到服务器
ssh root@47.79.232.189

# 3. 添加公钥
mkdir -p ~/.ssh
echo "你复制的公钥内容" >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
exit

# 4. 测试连接
ssh root@47.79.232.189
```
