# 🚀 Node 服务远程启动完整指南

## 📋 快速启动（复制粘贴执行）

### 方式 1：单条命令执行（推荐）

打开新的终端窗口，复制以下整段命令并粘贴执行：

```bash
ssh root@47.79.232.189 "cd /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master && pm2 restart all && sleep 2 && pm2 list && echo '===健康检查===' && curl -s http://127.0.0.1:8888/api/health"
```

**预期输出：**
- 看到 4 个进程状态都是 `online`
- 健康检查返回：`{"status":"ok","db":"connected","timestamp":...}`

---

### 方式 2：分步执行

#### 第 1 步：SSH 登录
```bash
ssh root@47.79.232.189
```
输入密码后回车

#### 第 2 步：进入项目目录
```bash
cd /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master
```

#### 第 3 步：查看当前状态
```bash
pm2 list
```

#### 第 4 步：重启服务
```bash
pm2 restart all
```

#### 第 5 步：等待 3 秒后查看状态
```bash
sleep 3 && pm2 list
```

#### 第 6 步：查看日志（可选）
```bash
pm2 logs bmm-api --lines 20 --nostream
```

#### 第 7 步：健康检查
```bash
curl http://127.0.0.1:8888/api/health
```

应该返回类似：
```json
{"status":"ok","db":"connected","timestamp":1732352437123}
```

#### 第 8 步：退出 SSH
```bash
exit
```

## ⚠️ 常见问题排查

### 问题 1：进程状态显示 `errored` 或 `stopped`

**解决方法：**
```bash
# 查看错误日志
pm2 logs bmm-api --err --lines 50

# 完全停止后重新启动
pm2 delete all
pm2 start /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master/server/pm2/ecosystem.config.js
```

### 问题 2：数据库连接失败

**检查数据库：**
```bash
# 测试数据库连接
mysql -u bitcoin_mining_master -pFzFbWmwMptnN3ABE -h 127.0.0.1 bitcoin_mining_master -e "SELECT 1"

# 检查 MySQL 服务状态
systemctl status mysql
```

### 问题 3：Redis 连接失败

**检查 Redis：**
```bash
# 测试 Redis 连接
redis-cli -h 127.0.0.1 -p 6379 -a 3hu8fds3y ping

# 检查 Redis 服务状态
systemctl status redis
```

### 问题 4：端口被占用

**检查端口：**
```bash
# 查看 8888 端口占用
netstat -tlnp | grep 8888
lsof -i:8888

# 如果被其他进程占用，可以杀掉
kill -9 <PID>
```

### 问题 5：健康检查返回 404 或 500

**排查步骤：**
```bash
# 1. 查看完整日志
pm2 logs --lines 100

# 2. 检查路由是否正确
curl -v http://127.0.0.1:8888/api/health

# 3. 重启服务
pm2 restart all

# 4. 如果还不行，完全重启
pm2 delete all
cd /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master
pm2 start server/pm2/ecosystem.config.js
```

## 📝 成功标志清单

✅ **服务启动成功的标志：**

1. `pm2 list` 显示所有进程状态为 `online`
2. 日志中有 "数据库已同步" 消息
3. 日志中有 "Server is running on port 8888" 消息
4. 健康检查返回 `{"status":"ok","db":"connected"}`
5. 没有 Redis 或数据库连接错误
6. 内存使用合理（每个进程 < 100MB）

## 🎯 一键健康检查脚本

在服务器上执行：

```bash
#!/bin/bash
echo "=========================================="
echo "       服务健康检查"
echo "=========================================="
echo ""
echo "1. PM2 进程状态："
pm2 list
echo ""
echo "2. API 健康检查："
curl -s http://127.0.0.1:8888/api/health | python -m json.tool 2>/dev/null || curl -s http://127.0.0.1:8888/api/health
echo ""
echo "3. 数据库连接："
mysql -u bitcoin_mining_master -pFzFbWmwMptnN3ABE -h 127.0.0.1 -e "SELECT 'OK' as status" 2>/dev/null && echo "✅ 数据库连接正常" || echo "❌ 数据库连接失败"
echo ""
echo "4. Redis 连接："
redis-cli -h 127.0.0.1 -p 6379 -a 3hu8fds3y ping 2>/dev/null && echo "✅ Redis 连接正常" || echo "❌ Redis 连接失败"
echo ""
echo "5. 端口监听："
netstat -tlnp | grep 8888 && echo "✅ 8888 端口正在监听" || echo "❌ 8888 端口未监听"
echo ""
echo "=========================================="
```
