# 云服务器 Node 服务启动指南

## 方式 1：SSH 登录后手动启动

```bash
# 1. SSH 登录到云服务器
ssh root@47.79.232.189

# 2. 进入项目目录
cd /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master

# 3. 查看当前 PM2 状态
pm2 list

# 4. 重启所有服务
pm2 restart all

# 5. 查看日志确认启动成功
pm2 logs bmm-api --lines 30

# 6. 测试健康检查
curl http://127.0.0.1:8888/api/health
```

## 方式 2：使用现有的重启脚本

```bash
# 在云服务器上执行
cd /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master/server/pm2
bash restart.sh
```

### 1. 检查进程状态
```bash

```
应该看到 4 个进程都是 `online` 状态：
- bmm-api (2 个实例，cluster 模式)
- bmm-worker (1 个实例)
- bmm-scheduler (1 个实例)

curl http://47.79.232.189/api/health

## 常见问题排查

### 问题 1：服务无法启动
```bash
# 查看错误日志
pm2 logs --err --lines 50

# 检查端口占用
netstat -tlnp | grep 8888

# 检查数据库连接
mysql -u bitcoin_mining_master -p -h 127.0.0.1
```

### 问题 2：数据库连接失败
检查 `/www/wwwroot/47.79.232.189/Bitcoin_Mining_Master/backend/.env` 文件：
```bash
cat backend/.env | grep -E "(DB_HOST|DB_USER|DB_PASSWORD)"
```

### 问题 3：Redis 连接失败
```bash
# 测试 Redis 连接
redis-cli -h 127.0.0.1 -p 6379 -a 3hu8fds3y ping
```

## 服务管理命令

```bash
# 重启所有服务
pm2 restart all

# 停止所有服务
pm2 stop all

# 查看实时日志
pm2 logs

# 查看特定服务日志
pm2 logs bmm-api

# 查看进程详情
pm2 show bmm-api

# 监控服务状态
pm2 monit

# 保存当前进程列表
pm2 save

# 设置开机自启
pm2 startup
pm2 save
```

## 注意事项

1. **环境变量**：确保 `.env` 文件中的数据库和 Redis 配置正确
2. **代码同步**：确保云服务器代码是最新的（已移除 rateLimiter）
3. **权限**：确保 PM2 进程有足够权限访问日志目录
4. **防火墙**：确保 8888 端口在防火墙中开放

## 验证 rateLimiter 已移除

```bash
# 搜索代码中是否还有 rateLimiter 引用
cd /www/wwwroot/47.79.232.189/Bitcoin_Mining_Master/backend
grep -r "rateLimiter" src/ --include="*.js"

# 应该只在注释中出现，不应该有实际调用
```

## 成功标志

启动成功后，你应该能看到：

1. PM2 列表中所有进程状态为 `online`
2. 日志中有 "数据库已同步" 和 "Server is running on port 8888" 消息
3. 健康检查返回 `{"status":"ok","db":"connected",...}`
4. 没有 Redis 或数据库连接错误

---

**重要提示**：由于服务需要访问本地数据库和 Redis，必须在云服务器上运行，不能在本地 Mac 运行。
