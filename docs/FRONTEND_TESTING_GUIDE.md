# 前端页面快速测试指南

## 🎯 测试目标

测试新创建的三个前端页面：
1. 积分中心页面 (PointsScreen)
2. 签到页面 (CheckInScreen)
3. 增强后的仪表盘 (DashboardScreen)

---

## 📋 测试前准备

### 1. 确保后端服务运行
```bash
# 在backend目录
cd backend
pm2 status

# 如果未运行，启动服务
pm2 start ecosystem.config.js
```

### 2. 确认API端点可访问
- 后端地址：http://localhost:8888
- Android模拟器访问：http://10.0.2.2:8888

### 3. 准备测试用户
- 需要有用户token（从登录/注册获取）
- 建议使用已有积分和签到记录的用户

## 🐛 常见问题排查

### 问题1: 页面显示"加载失败"

**可能原因**：
- 后端服务未启动
- Token未设置或已过期
- 网络连接问题

**解决方法**：
```bash
# 1. 检查后端服务
cd backend
pm2 status
pm2 logs bitcoin-mining-master

# 2. 检查API是否可访问
curl http://localhost:8888/api/points/balance -H "Authorization: Bearer YOUR_TOKEN"

# 3. 重新登录获取新token
```

### 问题2: 积分数据不显示或为0

**可能原因**：
- 新用户没有积分数据
- 数据库中没有记录

**解决方法**：
```bash
# 在backend目录，手动给用户添加积分
node
> const PointsService = require('./src/services/pointsService.js')
> await PointsService.addPoints('YOUR_USER_ID', 100, 'test', '测试积分')
```

### 问题3: 签到按钮不响应

**可能原因**：
- 今日已经签到
- API请求失败

**解决方法**：
- 检查按钮是否显示"今日已签到"（正常）
- 查看网络请求是否成功
- 检查后端日志

### 问题4: 日历显示异常

**可能原因**：
- intl包未正确安装
- 日期格式解析错误

**解决方法**：
```bash
cd mobile_client/bitcoin_mining_master
flutter pub get
flutter clean
flutter pub get
```
