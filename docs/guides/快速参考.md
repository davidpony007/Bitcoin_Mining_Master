# 🚀 快速参考卡片

## 方案对比

| 方案 | 优点 | 缺点 | 适合场景 |
|------|------|------|----------|
| **Git + GitHub** ⭐ | • 专业版本控制<br>• 完整历史记录<br>• 多人协作<br>• 免费私有仓库 | • 需要学习Git<br>• 需要网络 | 专业开发<br>长期项目 |
| **云盘同步** | • 零学习成本<br>• 自动同步<br>• 简单直接 | • 可能文件冲突<br>• 同步慢<br>• 无版本控制 | 个人项目<br>偶尔切换 |
| **远程开发** | • 无需同步<br>• 统一环境<br>• 可多端访问 | • 需要服务器<br>• 依赖网络<br>• 学习成本高 | 有服务器<br>专业团队 |

## 🎯 推荐方案: Git + GitHub

### 第一台电脑（当前Mac）

```bash
# 1. 一键初始化
cd "/Users/davidpony/Desktop/Bitcoin Mining Master"
./git-setup.sh

# 或手动执行
git init
git add .
git commit -m "Initial commit"

# 2. 创建 GitHub 私有仓库
# 访问: https://github.com/new
# 仓库名: bitcoin-mining-master
# 类型: Private (私有)

# 3. 推送代码
git remote add origin https://github.com/你的用户名/bitcoin-mining-master.git
git branch -M main
git push -u origin main
```

### 第二台电脑

```bash
# 1. 克隆项目
git clone https://github.com/你的用户名/bitcoin-mining-master.git
cd bitcoin-mining-master

# 2. 配置环境变量
cd backend
cp .env.example .env
# 编辑 .env，填入以下配置:
# DB_HOST=47.79.232.189
# DB_USER=bitcoin_mining_master
# DB_PASSWORD=FzFbWmwMptnN3ABE
# DB_NAME=bitcoin_mining_master
# REDIS_HOST=47.79.232.189
# REDIS_PASSWORD=3hu8fds3y
# PORT=8888

# 3. 安装依赖
npm install

# 4. 启动项目
npm start  # 或 npm run dev
```

## 📝 日常开发流程

### 在电脑 A 修改代码后

```bash
git add .
git commit -m "描述你的修改"
git push
```

### 在电脑 B 获取最新代码

```bash
git pull
```

### 如果遇到冲突

```bash
# Git会提示哪些文件冲突
# 手动编辑文件，解决冲突标记
# <<<<<<< HEAD
# 你的代码
# =======
# 别人的代码
# >>>>>>> 分支名

# 解决后
git add .
git commit -m "解决冲突"
git push
```

## ⚠️ 重要提醒

### 不要上传的文件（已配置 .gitignore）

```
✅ 已排除（安全）:
  • node_modules/          (依赖包，太大)
  • .env                   (敏感信息！)
  • logs/                  (日志文件)
  • build/                 (构建产物)
  • .DS_Store              (系统文件)
```

### 环境变量文件（.env）

**包含敏感信息，每台电脑需手动创建！**

```bash
# 后端配置
cd backend
cp .env.example .env

# 填入以下内容:
DB_HOST=47.79.232.189
DB_USER=bitcoin_mining_master
DB_PASSWORD=FzFbWmwMptnN3ABE
DB_NAME=bitcoin_mining_master
REDIS_HOST=47.79.232.189
REDIS_PORT=6379
REDIS_PASSWORD=3hu8fds3y
REDIS_KEY_PREFIX=bmm:
JWT_SECRET=your_jwt_secret_key_here_change_in_production
JWT_EXPIRES_IN=24h
PORT=8888
NODE_ENV=development
```

## 🔧 常用 Git 命令

```bash
# 查看状态
git status

# 查看修改内容
git diff

# 查看提交历史
git log --oneline

# 撤销修改（未提交）
git checkout -- 文件名

# 撤销提交（已提交）
git reset --soft HEAD~1

# 查看远程仓库
git remote -v

# 拉取最新代码
git pull

# 推送代码
git push

# 创建分支
git checkout -b 新分支名

# 切换分支
git checkout 分支名

# 合并分支
git merge 分支名

# 查看所有分支
git branch -a
```

## 🆘 常见问题

### Q: 忘记创建 .env 文件，项目启动失败

```bash
cd backend
cp .env.example .env
# 编辑 .env，填入配置
npm start
```

### Q: node_modules 文件夹没有

```bash
# 重新安装依赖
npm install
```

### Q: Git 提示 "failed to push"

```bash
# 先拉取远程最新代码
git pull

# 如果有冲突，解决后再推送
git push
```

### Q: 数据库连接失败

```bash
# 检查 .env 配置
cat backend/.env

# 确认数据库服务器可访问
ping 47.79.232.189

# 测试数据库连接
mysql -h 47.79.232.189 -u bitcoin_mining_master -p
```
