# 📱 多电脑同步开发指南

## 🎯 目标
在多台电脑上同时打开和编辑 Bitcoin Mining Master 项目代码

---

## 方案一：使用 Git + GitHub/GitLab（推荐）⭐

### 优点
- ✅ 版本控制，不怕代码丢失
- ✅ 多人协作友好
- ✅ 有完整的历史记录
- ✅ 可以回滚到任何版本
- ✅ 免费（公开仓库）或私有仓库

### 实施步骤

#### 1. 在当前电脑（Mac）初始化 Git

```bash
cd "/Users/davidpony/Desktop/Bitcoin Mining Master"

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: Bitcoin Mining Master 项目"
```

#### 2. 创建 .gitignore 文件（重要！）

项目根目录需要添加以下内容到 `.gitignore`：

```
# 依赖
node_modules/
*/node_modules/

# 环境变量（敏感信息不要上传）
.env
*/.env

# 日志文件
logs/
*.log
nohup.out
pm2-*.log

# 构建产物
build/
dist/
*/build/
*/dist/

# Android
*.apk
*.aab
local.properties
.gradle/
mobile_client/Bitcoin_Mining_Master/build/
mobile_client/Bitcoin_Mining_Master/app/build/
mobile_client/Bitcoin_Mining_Master/.gradle/

# iOS
*.xcworkspace
*.xcuserdata
Pods/
ios_client/Pods/

# 编辑器
.vscode/
.idea/
*.swp
*.swo
*~

# 系统文件
.DS_Store
Thumbs.db

# PM2
.pm2/
```

#### 3. 推送到 GitHub（私有仓库推荐）

```bash
# 在 GitHub 创建新仓库后（不要初始化 README）
git remote add origin https://github.com/你的用户名/bitcoin-mining-master.git

# 推送代码
git branch -M main
git push -u origin main
```

#### 4. 在另一台电脑克隆项目

```bash
# Windows/Mac/Linux
cd 你想存放项目的目录

# 克隆项目
git clone https://github.com/你的用户名/bitcoin-mining-master.git

# 进入项目
cd bitcoin-mining-master
```

#### 5. 配置环境文件

**⚠️ 重要**：`.env` 文件不会被同步（包含敏感信息），需要手动创建：

```bash
# 后端
cd backend
cp .env.example .env

# 编辑 .env，填入数据库和 Redis 配置
# 使用和当前电脑相同的云服务器配置
```

#### 6. 安装依赖

```bash
# 后端依赖
cd backend
npm install

# 前端依赖（如果需要）
cd ../web_frontend
npm install
```

#### 7. 日常开发流程

**在电脑 A 修改代码后：**
```bash
git add .
git commit -m "描述你做的修改"
git push
```

**在电脑 B 获取最新代码：**
```bash
git pull
```

## 方案二：使用云同步盘（简单但有风险）⚠️

### 适用场景
- 只有你一个人开发
- 不需要版本控制
- 两台电脑不会同时修改同一个文件

### 推荐工具
1. **iCloud Drive** (Mac 生态最好)
2. **Dropbox**
3. **Google Drive**
4. **OneDrive**
5. **坚果云**（国内速度快）

### 实施步骤

#### 1. 移动项目到云盘目录

**Mac 示例（iCloud）：**
```bash
# 移动项目到 iCloud
mv "/Users/davidpony/Desktop/Bitcoin Mining Master" \
   "/Users/davidpony/Library/Mobile Documents/com~apple~CloudDocs/Bitcoin Mining Master"

# 创建软链接（可选，保持原路径可用）
ln -s "/Users/davidpony/Library/Mobile Documents/com~apple~CloudDocs/Bitcoin Mining Master" \
      "/Users/davidpony/Desktop/Bitcoin Mining Master"
```

**Windows 示例（OneDrive）：**
```powershell
# 项目放到 OneDrive 目录
C:\Users\你的用户名\OneDrive\Bitcoin Mining Master
```

#### 2. 等待同步完成

在状态栏查看同步进度（iCloud 图标/OneDrive 图标）

#### 3. 在另一台电脑打开

等待云盘同步完成后，直接打开项目文件夹

#### 4. 注意事项

⚠️ **风险提示：**
- ❌ 不要同时在两台电脑修改同一个文件
- ❌ 不要在同步未完成时编辑代码
- ❌ `node_modules/` 文件夹会很大，同步慢
- ❌ 可能出现文件冲突

**建议排除的文件夹：**
- `node_modules/`（在新电脑重新 `npm install`）
- `build/`
- `.gradle/`
- `logs/`

## 方案三：使用远程开发（专业方案）🚀

### VS Code Remote Development

#### 1. 准备云服务器

你已经有了云服务器（47.79.232.189），可以：

```bash
# 在云服务器上克隆代码
ssh root@47.79.232.189
cd /home
git clone https://github.com/你的用户名/bitcoin-mining-master.git
```

#### 2. 本地安装 VS Code 插件

- 安装 `Remote - SSH` 插件
- 配置 SSH 连接到服务器
- 直接在 VS Code 中编辑服务器上的代码

#### 3. 优点

- ✅ 所有电脑都编辑同一份代码（云端）
- ✅ 不需要同步
- ✅ 代码和数据库在同一台服务器，调试方便
- ✅ 可以直接看到线上效果

## 🆘 常见问题

### Q1: 忘记添加 .gitignore，已经把 node_modules 推送了怎么办？

```bash
# 从 Git 中删除（但保留本地文件）
git rm -r --cached node_modules
git commit -m "Remove node_modules from git"
git push
```

### Q2: 两台电脑的代码冲突了怎么办？

```bash
# 拉取最新代码时遇到冲突
git pull

# Git 会提示冲突文件
# 手动编辑文件，解决冲突
# 然后：
git add .
git commit -m "解决冲突"
git push
```

### Q3: 云盘同步太慢怎么办？

- 排除 `node_modules/`、`build/` 等大文件夹
- 使用国内云盘（坚果云）
- 考虑使用 Git 方案
