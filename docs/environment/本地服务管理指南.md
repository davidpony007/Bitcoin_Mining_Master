## 管理脚本

项目根目录提供了以下管理脚本：

### 1. `check-services.ps1` - 检查服务状态
```powershell
.\check-services.ps1
```
显示所有服务的安装状态和运行状态。

### 2. `start-services.ps1` - 启动服务
```powershell
.\start-services.ps1
```
自动启动 Redis 服务。MySQL 需要单独启动（见下文）。

### 3. `start-mysql.ps1` - 启动 MySQL
```powershell
.\start-mysql.ps1
```
在新窗口启动 MySQL 服务器（需要保持窗口打开）。

或者在后台启动：
```powershell
Start-Process powershell -ArgumentList "-NoExit", "-File", "$PWD\start-mysql.ps1" -WindowStyle Hidden
```

### 4. `stop-services.ps1` - 停止服务
```powershell
.\stop-services.ps1
```
停止所有正在运行的 Redis 和 MySQL 服务。

## 常见问题

### Q: PATH 环境变量未生效？
A: 关闭并重新打开 PowerShell 窗口，或运行：
```powershell
$env:PATH = "$env:USERPROFILE\scoop\shims;$env:PATH"
```

### Q: Redis/MySQL 命令找不到？
A: 确保 Scoop 的 shims 目录在 PATH 中。运行上面的命令添加到当前会话。

### Q: MySQL 启动失败？
A: 检查是否有其他 MySQL 实例正在运行，或查看错误日志：
```powershell
Get-Content $env:USERPROFILE\scoop\persist\mysql-lts\data\*.err | Select-Object -Last 50
```

### Q: 如何设置 MySQL root 密码？
A: 连接 MySQL 后执行：
```sql
ALTER USER 'root'@'localhost' IDENTIFIED BY '你的密码';
FLUSH PRIVILEGES;
```

## 卸载服务

如果需要卸载：
```powershell

# 卸载
scoop uninstall redis mysql-lts
```
